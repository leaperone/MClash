# MClash 1.1.6

This patch release turns routing telemetry into a clearer, verifiable operating picture and strengthens App Routing's TCP data path.

## Traffic control and history

- App Routing now owns and relays matched TCP **Direct** flows instead of handing them back immediately, so their delivered upload and download bytes can be measured exactly.
- Added per-rule observed matches, active flows, failures, measured traffic, and last-match time in App Routing.
- Traffic History now shows both the App Routing rule and the root-to-exit Mihomo route for each completed session flow.
- Added optional private local aggregates for Today and This Week, including measured bytes, completed flows, measurement coverage, top applications, and top routes, with 7, 30, or 90 day retention.
- Persistent history is opt-in and stores no destination host, IP, port, PID, UID, executable path, connection identifier, raw error, or credential. Corrupt or newer databases are left untouched and reported as unavailable.

## Truthful live state

- Traffic, connection, proxy, and App Routing channels now expire after their expected cadence. A silent stream is shown as waiting, reconnecting, or stale instead of a credible live zero.
- Overview, Traffic, the sidebar, proxy inspector, and menu bar now gate rates, totals, connection counts, and relay counts on source freshness.
- Proxy evidence now says exactly what was observed: a non-direct route reported by Mihomo. It no longer claims that capture itself was proven by an inferred connection.
- Route chains are displayed consistently from rule root to exit node.
- Direct and fail-open payloads that remain outside MClash are explicitly labeled unmeasured; rejected flows remain no-payload rather than false zero-byte traffic.

## Verifiable operations

- Live System Proxy setting changes are transactional: apply, read back, verify, then save. A rejected change restores and verifies the previous settings, while a failed rollback becomes a durable Attention item.
- Settings shows the last verified System Proxy and App Routing provider state plus the outcome of the latest System Proxy settings change.
- Profile batch refreshes and provider update/health-check operations now retain visible success, unchanged, and failure receipts.
- Rules distinguish a real zero hit count from unavailable data, show the last hit, and refresh while the workspace is visible.
- Provider, rule, proxy latency, and log views now include the time associated with the displayed state.
- Attention recovery actions show in-progress state and block duplicate retries.

TCP Direct traffic is measured only when MClash owns the relay. UDP Direct, security bypasses, and true fail-open handoffs remain intentionally pass-through and explicitly unmeasured.
