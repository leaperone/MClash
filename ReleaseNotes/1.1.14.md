# MClash 1.1.14

This patch makes Proxifier `.ppx` profiles selectable in App Routing and hardens release packaging.

## Proxifier import

- Register `.ppx` as an imported Proxifier Profile type in the signed app bundle.
- Use the exact same declared type in the macOS file importer, preventing valid profiles from appearing greyed out.
- Keep the imported profile fully compatible with domain groups, CIDR ranges, applications, processes, ports, protocols, and actions supported by the existing importer.
- Add release-packaging coverage so the bundle declaration and Swift importer type cannot silently drift apart.

## Release reliability

- Replace the Finder-dependent writable-DMG flow with a direct, verified read-only image build.
- Remove the headless Finder automation and mount-point collision that caused the unpublished 1.1.13 packaging run to fail.

## Verification

- Verified the provided `Untitled.ppx` profile remains fully parseable after the file-type fix.
- Verified the generated DMG contains the application, the Applications shortcut, a valid code signature, and the declared `.ppx` type.
- Verified 259 application tests and 101 shared networking tests, followed by warnings-as-errors direct links of the host and Network Extension.
