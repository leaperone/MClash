# MClash 1.1.11

This patch makes App Routing rules faster to create and keeps the live activity view focused on traffic that needs attention.

## App Routing rules

- Make the application optional so a domain-only or IP/CIDR-only rule applies across applications, while preserving MClash's built-in safety bypasses.
- Move application, domain, IP, and CIDR matching into the main rule workflow instead of hiding destinations under Advanced Matching.
- Support multiple domains, IP addresses, and CIDR networks in one rule with comma, space, semicolon, or newline-separated input, normalization, and duplicate removal.
- Treat destinations within a rule as alternatives and combine an optional application with the destination group, enabling rules such as `OpenAI App AND (openai.com OR oaistatic.com)`.
- Preserve and edit existing multi-destination rules instead of rejecting them as unsupported.

## Activity

- Default to a low-noise `Proxy & Issues` view that hides normal Direct traffic without discarding it.
- Keep Mihomo routes, rejects, fail-open handoffs, relay failures, and Mihomo-to-Direct fallbacks visible by default.
- Retain explicit All and Direct filters, show both visible and retained flow counts, and provide a one-click reveal when only Direct traffic exists.

## Verification

- Added coverage for domain-only rules, application-and-destination matching, multi-domain groups, mixed IP/CIDR destinations, fallback visibility, and Direct filtering.
- Verified 249 application tests and 99 shared networking tests, followed by warnings-as-errors direct links of the host and Network Extension.
