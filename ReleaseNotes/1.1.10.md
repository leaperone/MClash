# MClash 1.1.10

This patch fixes App Routing system extension installation on macOS.

## Fixes

- Restore the shared macOS App Group entitlement required as the prefix of `NEMachServiceName`; its removal caused Network Extension category validation to reject the 1.1.9 system extension.
- Keep host-to-provider runtime control on Apple's authenticated `NETunnelProviderSession.sendProviderMessage` channel rather than relying on a shared-container heartbeat.
- Preserve the verified App Routing and DNS Routing coupled lifecycle, rollback, runtime telemetry, and CPU fixes introduced in 1.1.9.

## Verification

- Release packaging now verifies the App Group in both final signatures and rejects a Mach service that is not its child.
- Added regression coverage for the shared App Group, Mach service prefix, and both provider class registrations.
