# MClash 1.1.21

This patch makes startup reliable, connects usable profiles automatically, and turns the menu bar into a quieter everyday control surface.

## Startup and Connection

- Restore the system-managed main-app login item and migrate the previous background-agent registration safely.
- Enforce one MClash instance per user, including launches from another app copy or an outdated login item.
- Start application preparation even when the main window stays hidden at login.
- Connect the active profile automatically and enable App Routing whenever no preference has been saved; explicit opt-outs remain respected.
- Show a direct shortcut to macOS Login Items when login launch is waiting for approval.

## Menu Bar and Quit

- Ask whether to keep MClash and its proxy services running in the menu bar or quit completely when the user quits the app.
- Let users choose between the MClash logo and live download, upload, and connection status in the menu bar.
- Remove redundant descriptions and secondary controls from the menu-bar panel while keeping connection, profile, App Routing, settings, and recovery actions close at hand.
- Avoid unnecessary background telemetry when the logo-only menu-bar style is selected.

## Safety and Verification

- Preserve the mutual exclusion between App Routing and the macOS System Proxy across automatic, manual, and restarted connections.
- Add regression coverage for single-instance locking, login-item launch detection, background quit behavior, default App Routing, persistent opt-out, menu-bar telemetry, and release packaging.
- Verify warnings-as-errors type checking, direct unit tests, and the full integration suite before release.
