# MClash 1.1.17

This patch adds a secure local automation API for AI agents and external tools, while preserving the lightweight menu-bar resident lifecycle introduced in 1.1.16.

## Automation

- Add the bundled `mclashctl` command-line client and a versioned JSON-RPC API over a private Unix domain socket.
- Expose application lifecycle, updates, settings, core control, profiles, backup, runtime overrides, routing, providers, System Proxy, App Routing, Proxifier import, traffic history, logs, and diagnostics.
- Publish machine-readable capability metadata with parameter types, scopes, risk, pagination limits, and interaction requirements.
- Start MClash invisibly when invoked by automation, and keep the menu-bar service active without mounting the main SwiftUI window.
- Add `--params-stdin`, `--params-file`, and stable `--request-id` support for secret-safe and idempotent agent workflows.

## Security and reliability

- Bind pairings to the signed client identity, reject unsigned clients, store host authorization records and CLI tokens in separate Keychain access groups, and require local confirmation for destructive commands.
- Redact credentials and sensitive URLs, reject arbitrary file-path access, strictly validate parameters and page bounds, and cap frames, connections, caches, and response sizes.
- Make mutations idempotent across token and scope rotation, preserve stable client identities, roll back failed profile creation/activation transactions, and distinguish same-execution queries from new retries.
- Add absolute socket I/O deadlines, active-client shutdown, interaction rate limits, bounded candidate discovery, and paged compact receipts for large data sets.

## Background efficiency

- Keep login-item launches and automation launches windowless through the bundled background LaunchAgent.
- Precompute Flow Ledger aggregates and completed-entry indexes on the detached ledger worker so menu-bar operation and external polling do not recreate main-thread full scans.
- Keep expensive connection, log, proxy, and App Routing presentation streams dormant until a visible surface requests them.

## Verification

- Verified 283 host-application tests, 105 shared networking tests, and 4 automation protocol tests.
- Verified direct linking of MClash, `mclashctl`, and the Network Extension, plus Mihomo, HTTP/SOCKS, crash-recovery, System Proxy, and API integration smoke tests.
