# MClash 1.3.0 RC 3

This release candidate focuses on a quieter, lighter, and more flexible daily workflow.

## Highlights

- Adds quiet launch at login, enabled by default, without opening or focusing the main window.
- Adds Lightweight Mode to suspend background connection-history polling and dynamic menu-bar status work when no relevant surface is visible.
- Restores the last explicit Connect or Disconnect intent after relaunch and improves App Routing recovery after a core crash or Mac restart.
- Adds simultaneous multi-profile sessions. Every enabled profile runs in an isolated Mihomo process with its own runtime configuration, core home, controller secret, and Mixed port.
- App Routing rules can now target an exact profile and choose that profile's Rules or GLOBAL route. Policy-group selection remains available for the current default profile in this RC.
- Simplifies local proxy configuration to one Mixed port per profile. MClash no longer exposes separate HTTP and SOCKS listener settings; the Mixed port accepts both protocols.
- Isolates every managed session to its assigned Mixed port and private App Routing listeners. Profile-owned custom listeners, TUN, tunnels, server shortcuts, and external controllers are not launched; default-profile Redirect, TProxy, and DNS settings remain available.
- Adds profile-aware listener catalogs and exact TCP/UDP routing so traffic for one profile cannot silently use another profile's private endpoint.
- Adds duplicate-port validation, unavailable-profile fallback, profile deletion safeguards, cold-provider reconciliation, and bounded per-profile crash recovery.
- Makes release verification deterministic by explicitly disabling in-process test parallelism for Apple manager integration tests.

## Testing focus

Please test at least two profiles at once:

1. Assign development tools to Profile A.
2. Assign a browser and Telegram to Profile B.
3. Verify both HTTP and SOCKS clients work on each profile's Mixed port.
4. Restart the Mac with quiet login and Lightweight Mode enabled.
5. Confirm both profile assignments, ports, App Routing, DNS status, and the last connection intent are restored.

This is a prerelease build intended for hands-on validation before MClash 1.3.0.
