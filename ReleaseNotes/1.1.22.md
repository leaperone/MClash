# MClash 1.1.22

This patch makes Sparkle updates substantially smaller while preserving the existing full-package safety path.

## Incremental Updates

- Publish verified Sparkle delta updates from up to two recent stable builds when they are smaller than the full update archive.
- Reverse-apply every generated delta and verify the reconstructed app's complete code signature before adding it to the update feed.
- Keep the notarized full ZIP in every release so unsupported source builds and failed delta applications automatically fall back to the complete update.
- Include delta assets in release checksums and GitHub Release publication without weakening immutable versioned download URLs.

## Update Reliability

- Let Sparkle's install-and-relaunch flow bypass the normal interactive quit choice while still stopping networking services and restoring macOS proxy settings safely.
- Support both Sparkle XML namespace variants when attaching delta metadata to generated appcasts.
- Add release-pipeline regression tests and document the delta generation, verification, and fallback behavior.
