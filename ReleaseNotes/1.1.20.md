# MClash 1.1.20

This patch keeps established App Routing connections alive during compatible rule edits and turns Activity into a live connection view.

## App Routing

- Apply rule-only configuration changes to the running Network Extension without stopping Mihomo or disconnecting established managed flows.
- Persist updated rules before delivering them to the active provider so the saved and running configurations stay aligned.
- Fall back to a controlled restart only when a change requires a new listener topology or DNS configuration.
- Explain in the rule editor when a change can be applied live and when a restart is required.

## Activity

- Replace the retained event-log presentation with a stable table of currently managed connections.
- Show each connection's target, route and state, live upload and download speed, transferred bytes, and duration.
- Keep idle connections visible at zero speed and remove rows after their flows complete or fail.
- Clarify that ordinary Direct flows return to macOS and cannot be observed continuously by MClash.

## Verification

- Add regression coverage for live rule updates, preserved DNS runtime identity, structural restart fallback, active-flow classification, and per-flow traffic rates.
- Verify warnings-as-errors type checking, unit tests, and the full integration suite before release.
