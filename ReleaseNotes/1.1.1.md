# MClash 1.1.1

This patch release makes per-application routing easier to discover and configure.

## Highlights

- Added a dedicated **App Routing** destination to the main sidebar.
- Replaced the settings form with a Proxifier-style ordered rule table.
- Added rule duplication, enable/disable, deletion, and move-up/move-down controls.
- Simplified the common workflow to application → action → save, with technical conditions folded into **Advanced Matching**.
- Added **Choose Other Application…** so a signed `.app` can be selected from Finder even when it is not running.
- Added application icons, bundle identifiers, executable details, readable rule names, and a natural-language rule preview.
- Added `*` and `?` wildcard matching for application names, bundle identifiers, signing identifiers, and executable names.
- Added `*.example.com` domain wildcard input, normalized to secure domain-and-subdomain matching.
- Kept exact application rules secured by the selected app's designated code-signing requirement.
- Kept exact process, executable path, UID, IP, CIDR, TCP/UDP, port-range, Direct, Reject, Mihomo, and fallback controls available under advanced options.

## Usage

Open **App Routing** in the sidebar, choose **Add Rule…**, select a running application or **Choose Other Application…**, choose an action, and save. Rules are evaluated from top to bottom.
