# MClash 1.1.19

This patch restores native Direct traffic performance and reduces UI and background hot-path overhead throughout MClash.

## Direct Routing

- Return initial Direct and fail-open flows to macOS immediately instead of owning them for a second socket and user-space relay.
- Keep reject and Mihomo traffic inside the Network Extension while preserving Direct fallback for already-owned proxy flows.
- Resolve Mihomo availability per route so a missing listener falls back exactly as configured.
- Cache verified process identities and pre-index routing rules to reduce work on every new connection.

## Responsiveness

- Move connection, log, and App Routing activity filtering and presentation work off the main thread.
- Debounce high-frequency searches and coalesce live telemetry updates to avoid unnecessary SwiftUI redraws.
- Reduce hidden-view polling and skip background refresh work when its UI is not visible.
- Cache and invalidate network-service discovery instead of repeatedly invoking expensive system configuration commands.

## Verification

- Add regression coverage for Direct flow ownership, Mihomo route availability, process identity caching, and network-service cache invalidation.
- Verify warnings-as-errors type checking, unit tests, and the full integration suite before release.
