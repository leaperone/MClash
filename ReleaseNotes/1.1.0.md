# MClash 1.1.0

This minor release adds native per-application and per-process traffic capture backed by Apple's Network Extension framework and the bundled mihomo core.

## Highlights

- Added per-application rules secured by each app's designated code-signing requirement.
- Added exact running-process rules for GUI apps, command-line tools, and accessible background processes, guarded by PID, process start time, and canonical executable path.
- Added destination matching by IPv4/IPv6 address, CIDR network, domain, TCP/UDP protocol, and port or port range.
- Added Direct, Reject, and Mihomo profile-rule actions with explicit unavailable-route fallback behavior.
- Added a macOS transparent proxy system extension with real TCP SOCKS5 and UDP ASSOCIATE relays into mihomo.
- Added a private dual-stack, loopback-only mihomo listener protected by launch-scoped credentials.
- Added immutable loopback, link-local, multicast, and trusted MClash-component bypasses to prevent routing loops and local-network disruption.
- Added transactional extension activation, versioned rule updates, fail-open recovery, System Proxy mutual exclusion, and safe core shutdown ordering.
- Added complete Developer ID provisioning-profile validation and system-extension signing to the release workflow.

## Installation

Download `MClash-1.1.0-macos-arm64.dmg`, open it, and drag MClash to Applications. Existing installations can update through **MClash > Check for Updates…**.

The first time per-application capture is enabled, macOS asks you to approve the MClash system extension in System Settings. A restart may be required if macOS reports that the extension activation is pending reboot.

This release supports Apple Silicon Macs running macOS 14 or later. Intel Macs are not supported yet.
