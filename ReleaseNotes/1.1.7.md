# MClash 1.1.7

This patch release completes the App Routing data plane and makes its decisions, traffic, health, and operational effects visible before and after every change.

## Complete routing paths

- UDP flows now have managed relay sessions with per-destination upstream state, backpressure, resource limits, exact Direct delivery accounting, and route re-evaluation.
- Rules can explicitly use the active profile, Mihomo GLOBAL, or a named policy group. MClash creates isolated private Mihomo listeners for every configured route target instead of silently falling back to profile rules.
- macOS DNS traffic now has a real TCP and UDP proxy data plane through Mihomo, including provider heartbeat, SOCKS association, query-forwarded, and response-delivered evidence.
- DNS health probes tolerate transient failures, stop during sleep, recover after wake, and restore normal system DNS if the provider or backend can no longer be verified.

## Control and evidence

- Activity now explains exactly why a rule matched: signed application identity, identifier or path, process instance, user, destination, protocol, and port evidence are shown without retaining audit tokens, requirements, hashes, payloads, or credentials.
- Exact relay-port correlation is distinguished from probable destination/time correlation throughout Activity, Connections, and the flow inspector.
- Live measured App Routing rates are shown alongside Mihomo rates, including Direct traffic, per-rule traffic, per-application traffic, per-path traffic, retained-record coverage, and dropped-record counts.
- Long-lived active flows remain counted after the recent-outcome window, and measured Direct relays are no longer mislabeled as unobservable macOS handoffs.

## Safer operations

- Enabling App Routing now previews that Mihomo will restart, current connections can close, macOS may request Network Filter approval, and an active MClash System Proxy will be turned off.
- App Routing configuration changes are transactional. On failure, MClash restores the saved rules, listener configuration, active profile, core connection, and previously enabled System Proxy; incomplete rollback remains in Attention.
- DNS failures and automatic DNS restoration are visible in Attention with a Retry action.
- System Proxy Guard can be paused without turning off the current proxy. Settings explains its interaction with tools such as Proxifier and shows verification time, repair count, and the latest repaired external change.

The App Routing workspace remains native to the main sidebar and now provides separate Rules and Activity views, application selection from installed `.app` bundles, advanced destination/protocol/port matching, route actions, live status, operation receipts, and detailed flow inspection.
