# MClash 1.1.16

This patch makes MClash a lightweight menu-bar resident app when its main window is closed and removes the high-CPU presentation work found during runtime profiling.

## Background operation

- Closing the main window now unloads its SwiftUI content while keeping MClash, mihomo, System Proxy protection, Network Extension health checks, and the menu-bar item running.
- Opening MClash from the menu bar, Settings command, Dock, or a subscription URL restores the same main window and its last selected destination.
- Closing, minimizing, or hiding the main window suspends its presentation tree without stopping the resident proxy service.
- Closing the final window never quits MClash; Quit and Command-Q continue to use the safe shutdown path that restores network settings.
- The menu-bar label observes only durable core and capture state, instead of invalidating the entire application scene for live telemetry samples.

## CPU and memory usage

- Run high-frequency traffic, connection, log, proxy, and App Routing presentation work only while the selected page or open menu-bar panel needs it.
- Keep bounded five-second connection and App Routing accounting active in the background so session and persistent traffic history do not develop gaps.
- Drain mihomo process pipes without decoding or publishing each log line while the Logs page is unmounted.
- Collapse paged App Routing updates into one published merge rather than repeatedly sorting and rebuilding for every 250-record page.
- Coalesce Flow Ledger updates behind one utility worker, preventing cancelled generations from continuing concurrently.
- Index Flow Ledger connection matching by normalized destination, port, and exact relay source port, removing the previous activity-by-connection full scan.
- Bound WebSocket buffers so a busy UI cannot accumulate unlimited traffic, connection, or log snapshots.
- Make main-window registration idempotent so frequent model updates no longer cause repeated AppKit minimum-size layout work.

## Verification

- Runtime profiling of the previous installed build showed the MClash host at roughly 39–54% CPU with no visible windows while mihomo used roughly 2–4%; samples identified retained SwiftUI layout and overlapping Flow Ledger workers as the dominant paths addressed here.
- Verified 270 host-application tests and 105 shared networking tests.
- Verified warnings-as-errors type checking and direct linking of the host application and Network Extension.
