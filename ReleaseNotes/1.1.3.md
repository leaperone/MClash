# MClash 1.1.3

This patch release fixes App Routing system-extension installation and makes every activation step visible.

## Fixes

- Fixed the Developer ID signing entitlements so macOS accepts the Network Extension's Mach service and presents the system approval flow.
- Added release-time checks that require the host app and Network Extension to share the exact Team-ID App Group.
- Show the exact system-extension, network-filter, or DNS-proxy error instead of a generic “Needs Attention” status.
- Added persistent approval guidance with a button that opens the relevant System Settings area.
- Added explicit waiting-for-connection, approval-required, restart-required, and activation-failed states.
- Added a one-click activation retry and a direct link to MClash logs after a failure.
- Start MClash automatically when App Routing is enabled while disconnected.
- Prevent duplicate Network Extension activation requests during core startup.

## Enabling App Routing

Turn on **App Routing**. When macOS requests approval, open **System Settings → General → Login Items & Extensions → Network Extensions** and enable MClash. The App Routing page remains on the approval state until macOS completes the request.
