# MClash 1.1.4

This patch release fixes App Routing traffic interception and adds live, per-flow routing visibility.

## Fixes

- Fixed the Network Extension's running-code validation flags. The previous build received `errSecCSInvalidFlags` for every process and failed open instead of applying App Routing rules.
- Bypass MClash and its bundled Mihomo process using validated Team ID and exact signing identifiers, preventing recursive interception of the proxy core's own traffic.
- Verify the provider's running state, capture state, and configuration revision before reporting App Routing as enabled.
- Show unsupported forced-route fallbacks and SOCKS relay failures as actual outcomes instead of presenting them as successful proxy routes.

## App Routing Activity

- Added a dedicated **Activity** workspace beside **Rules** in App Routing.
- See the originating application and PID, destination, protocol, matched App Routing rule, effective action, relay state, start time, and upload/download totals.
- Active and recently closed flows are associated with Mihomo's rule and node chain using the relay's exact local source port plus destination and start-time metadata.
- Activity uses a bounded in-memory buffer, cursor-based incremental updates, and exports no audit tokens, signing requirements, or proxy credentials.
