# MClash 1.2.1

This minor release makes MClash more resilient across everyday Mac network changes, makes routing behavior easier to verify, and adds faster daily controls.

## Network reliability

- Verify the Mihomo controller, HTTP/SOCKS listeners, private Network Extension listeners, App Routing provider revision, DNS runtime, and System Proxy after sleep, wake, or a meaningful network-path change.
- Recover only when a connected session needs it, debounce rapid changes, and bound retries to prevent restart storms.
- Cancel pending recovery during sleep, intentional disconnects, and application shutdown.

## App Routing and DNS clarity

- Show the effective policy above App Routing rules: applications are explicitly Direct when no rule is enabled, DNS shows its independent On/Off state, and populated rules explain that the first match wins.
- Attribute DNS provider flows as **DNS Proxy** in Traffic and inspectors instead of presenting them as ordinary App Routing traffic.
- Preserve App Routing attribution for history written by earlier versions.

## Daily controls

- Pin up to three menu-bar Quick Routes; unfilled positions continue to follow profile order, and automatic ordering can be restored at any time.
- Add native keyboard commands for Overview, Traffic, Proxies, App Routing, Profiles, Connect/Disconnect, and Rule/Global/Direct modes.

## Subscription updates

- Persist refresh failure time, consecutive failure count, and next retry time without storing endpoint error text.
- Use jittered exponential automatic retry backoff from 15 minutes up to 24 hours while keeping manual refresh immediately available.
- Clear the retry state after a successful update or a valid not-modified response.

## Updates and verification

- Run Network Extension regression tests in the direct local test path as well as SwiftPM CI.
- Publish signed Sparkle delta updates for eligible recent builds while retaining the complete signed archive as the fallback.
