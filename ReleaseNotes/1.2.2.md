# MClash 1.2.2

This patch makes local automation practical for trusted workflows, adds eight
interface languages, and provides clearer guidance for App Routing and daily
operation.

## Local automation

- Offer two explicit pairing choices: **Allow Needed Access** keeps scoped
  access and one-time confirmation for destructive operations, while **Trust
  This Client** grants all scopes and permits unattended destructive operations
  for 180 days or until revoked.
- Preserve accumulated standard scopes across later approvals and keep trusted
  authority until the pairing is revoked.
- Fix the packaged `mclashctl` helper by removing restricted standalone CLI
  entitlements that could prevent it from launching, while continuing to store
  its token in the current user's Keychain.
- Keep the private mode-0600 Unix socket, same-user checks, code-identity
  binding, expiration, and revocation protections.
- Explain that trusting the bundled `mclashctl` also grants its authority to
  processes under the same macOS login that can invoke the helper.

## Languages

- Add English, Simplified Chinese, Traditional Chinese, Japanese, Korean,
  French, German, and Spanish interface localizations.
- Add a **System Default** option and an in-app language picker that updates the
  MClash interface without relaunching.
- Localize MClash-owned automation, quit, network recovery, and system-extension
  guidance, and package the localization bundles in local and release builds.

## App Routing guidance

- Document first-match behavior and a more predictable way to organize App
  Routing rules.
- Recommend selected signed applications over broad name patterns, one
  application or intent per rule, narrow exceptions before broad fallbacks, and
  UDP coverage for QUIC or call traffic.
- Expand the explanation of rule actions, fallback behavior, Proxifier imports,
  and the Activity and Traffic evidence available for checking actual routing
  decisions.

## Documentation and privacy

- Rewrite the README around installation, first use, automation, development,
  repository structure, releases, security, and support.
- Add a privacy document describing locally stored data, network requests,
  automation credentials, exports, backups, and removal.
- Expand automation documentation and manual release checks for both trust
  modes, CLI packaging, and all supported languages.
