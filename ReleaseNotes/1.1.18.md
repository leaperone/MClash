# MClash 1.1.18

This patch allows the MClash host application to use App Routing for GitHub updates, subscriptions, and other external requests without weakening the data-plane capture-loop protections.

## App Routing

- Remove the `one.leaper.mclash` host application from the built-in forced-Direct component list.
- Allow MClash update checks and other host requests to match application, domain, IP, protocol, and port rules like any other application.
- Continue forcing `mclash-mihomo` Direct so proxy egress can never be captured recursively.
- Continue forcing the MClash Network Extension Direct so relay and DNS provider traffic cannot re-enter App Routing.
- Preserve loopback, link-local, multicast, and unspecified-address bypasses.

## Verification

- Add regression coverage proving that MClash traffic to GitHub can select Mihomo while identical Mihomo core traffic remains on the built-in Direct bypass.
- Verify 283 host-application tests, 106 shared networking tests, and 4 automation protocol tests.
- Verify warnings-as-errors type checking and direct linking of MClash, `mclashctl`, and the Network Extension.
