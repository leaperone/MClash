# MClash 1.1.15

This patch makes App Routing rules easier to build and fixes Codex and other CLI traffic that could bypass matching when macOS process inspection was unavailable.

## App Routing rules

- Support multiple applications, running processes, and application or process identifiers in one rule.
- Move identifier entry to the top of the Sources editor and split pasted semicolon-, comma-, or newline-separated values into individually removable items.
- Support selecting several `.app` bundles at once and automatically include exact identifiers for app-owned CLI and helper executables.
- Keep source alternatives explicit: applications and processes use OR matching, while a configured source group and destination group use AND matching.
- Default newly selected applications to both TCP and UDP so QUIC and datagram traffic are not unintentionally omitted.

## Traffic capture reliability

- Continue evaluating destination-only and kernel-published application identifier rules when full process identity lookup fails, including permission-denied CLI processes.
- Preserve the validated destination hostname in SOCKS5 TCP and UDP requests to Mihomo so domain routing is not lost after macOS resolves the address.
- Retain the original IP endpoint for Direct routing, fallback, UDP conversation identity, and reply delivery.
- Add bounded per-flow diagnostics and user-visible activity notes when macOS flow metadata is used instead of complete process details.
- Preserve the built-in MClash component bypass when only kernel flow metadata is available, preventing self-capture loops.

## Verification

- Verified 263 application tests, 105 shared networking tests, and 5 Network Extension hostname-forwarding tests.
- Verified warnings-as-errors type checking and direct linking of both the host application and Network Extension.
