# MClash 1.1.2

This patch release fixes the App Routing page layout and responsiveness.

## Fixes

- Fixed a SwiftUI sizing regression that could push the sidebar, status header, rule rows, and action bar outside the visible window.
- Kept the App Routing page constrained to the actual detail-column size in both empty and populated rule states.
- Stabilized destination switching inside the macOS navigation split view.
- Moved running-application signature checks and process discovery off the main actor so opening or refreshing App Routing no longer blocks interface rendering.
- Added a visible refresh-in-progress state and cancellation-safe candidate updates.
