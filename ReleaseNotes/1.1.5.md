# MClash 1.1.5

This patch release makes MClash's routing state observable and actionable from end to end.

## Operational control

- Added one shared operating status across Overview, the sidebar, and the menu bar, separating the Mihomo Core from actual macOS traffic capture.
- Added a durable **Attention** workspace that keeps simultaneous Core, controller, System Proxy, App Routing, telemetry, rule, provider, and storage issues visible with their impact and recovery action.
- Added source freshness tracking. Reconnecting or stale traffic, connection, proxy, log, and App Routing data is labeled instead of being shown as a credible live zero.
- The menu bar now shows capture coverage, live rates, active connections, App Routing relays and rules, recent successful proxy evidence, and direct links to Overview, Traffic, App Routing, and Attention.

## Traffic visibility

- Renamed Connections to **Traffic** and added Live, Apps, Routes, and History workspaces.
- Added a unified flow ledger that correlates App Routing decisions with Mihomo connections and preserves application identity, capture origin, destination, App rule, Mihomo rule, root-to-leaf proxy path, lifecycle, and outcome.
- Added Top Applications and Top Routes to Overview, with observed traffic and active-flow counts.
- Direct and fail-open flows now say **Not measured after handoff** instead of reporting a false `0 B`; rejected flows say **No payload**.
- App Routing Activity now includes outcome filters, Provider heartbeat, per-minute totals, stable Mihomo correlation, and a full route-pipeline inspector.

## Reliability and diagnostics

- Continuously verify the Network Extension's actual running state and revision instead of trusting cached Host state. Provider drift and repeated heartbeat failure become an actionable failure.
- Surface Network Extension disconnect errors reported by macOS and safely stop persisted Provider configurations even after the Host restarts.
- System Proxy guard failures are retained, retried, and promoted to a recovery state after repeated verification failures; successful verification clears the issue.
- Storage initialization failures no longer appear as an empty healthy profile or settings list.
- Core shutdown no longer reports success while a process remains alive after termination attempts.
- Diagnostic export now includes operating state and live-source freshness while redacting recognized tokens, passwords, URL credentials, Bearer values, and controller secrets.
- Reduced App Routing telemetry overhead with constant-time activity lookup and rate-limited relay progress updates while preserving exact terminal totals.
