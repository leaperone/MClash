# MClash 1.1.9

This patch fixes DNS Routing startup and makes App Routing and DNS Routing operate as one verifiable data plane.

## Fixes

- Fixed DNS Provider startup rejecting a valid private Mihomo listener configuration.
- Replaced the heterogeneous DNS startup dictionary with one versioned, validated bootstrap payload that survives the NetworkExtension property-list boundary.
- Validate the complete saved DNS bootstrap—including activation identity and private relay—before reporting that DNS Routing is configured.
- Return a precise DNS bootstrap failure immediately instead of waiting 12 seconds for a heartbeat that can never appear.
- Relay DNS lifecycle, backend readiness, flow counts, and byte counters through the existing authenticated transparent-provider message channel.
- Force a verified DNS Provider off→on restart on every activation, including recovery from a previously failed enabled configuration.
- Bypass the private SOCKS listener only for cryptographically verified MClash/mihomo DNS egress, preventing a DNS→Mihomo→DNS recursion loop.
- Removed the cross-user App Group heartbeat file, which is not a valid bridge between a root system extension and its logged-in-user host app.
- Removed the unprofiled App Group entitlement that caused `trustd` to ignore restricted signing entitlements at runtime.
- Keep DNS Routing transactionally coupled to App Routing: both become operational together, and either startup failure rolls both back safely.
- Clear stale green DNS state after a failure and show the actual coupled data-plane error instead of leaving the UI on “Verifying Provider”.
- Fixed a main-thread CPU regression where the persistent sidebar repeatedly rescanned every live connection and rebuilt ISO 8601 formatters on each SwiftUI refresh.
- Cache non-direct route evidence in the background flow ledger, parse each connection timestamp once, and promptly cancel superseded ledger work when a newer connection snapshot arrives.

## Verification

- Added property-list bridge coverage for the versioned DNS bootstrap.
- Added complete bootstrap readback, forced off→on restart, activation-scoped runtime report, immediate failure, DNS opt-out cleanup, and coupled rollback coverage.
- Release packaging now rejects any reintroduction of the App Group entitlement that is absent from the current Developer ID profiles.
- Added route-evidence and timestamp parsing regression coverage for the CPU fix.
